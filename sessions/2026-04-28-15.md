## Apps Script bridge — redesign sketch (Step 5 of plan, surfaced here for review)

**Today (pre-migration):**

```
hypothesis_builder.html ──doPost──► Apps Script ──setValues──► Sheet
                                        │
                                        ▼
                                       (master register, 118 rows)
                                        │
                                        ▼
n8n WF-15 ──HTTP GET──► Apps Script doGet ──getValues──► Sheet
```

Single source of truth: Sheet. One read endpoint, one write endpoint, both in Apps Script.

**Post-migration target:**

```
hypothesis_builder.html ──doPost──► Apps Script doPost ──pg INSERT/UPDATE──► Postgres (canonical)
                                                                                 │
                                                                                 ▼
                                                            sync job (every N minutes or post-write)
                                                                                 │
                                                                                 ▼
                                                                            Sheet (derived view, read-only for humans)
                                                                                 │
                                                                                 ▼
n8n WF-15 ──HTTP GET──► Apps Script doGet ──pg SELECT──► Postgres
```

Two changes from the today picture: (1) Apps Script's doGet and doPost talk to Postgres instead of the Sheet directly; (2) a sync job exists that materialises Postgres into the Sheet for human review. Apps Script becomes a thin HTTP-to-PG adapter; the Sheet becomes a read-only mirror.

Minimal-change variant if Apps Script ↔ PG is too risky in Phase 1: keep doGet pointing at the Sheet, run the sync job PG → Sheet, and only the write path (doPost) talks to PG directly. n8n is unaware, no contract change.

This sketch is for review. No Apps Script code changes today.
